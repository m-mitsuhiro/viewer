## プロジェクト概要

**名前**: viewer
**種別**: デスクトップアプリケーション（ローカル画像・動画ビューア）
**対象ユーザー**: 個人ユーザー（自分のローカルファイルを快適に閲覧・整理したい人）
**主な目的**: ローカルに保存された大量の画像・動画ファイルを高速・快適に閲覧・管理する

---

## 対応ファイル形式

### 画像
- JPEG (`.jpg`, `.jpeg`)
- PNG (`.png`)
- WebP (`.webp`)
- BMP, TIFF など一般的なラスター形式も可能な範囲で対応

### 動画
- MP4 (`.mp4`)
- MKV (`.mkv`)
- AVI, MOV など一般的な動画形式も可能な範囲で対応

## 機能仕様

### ファイル管理・整理

| 機能 | 説明 |
|------|------|
| フォルダツリー表示 | サイドバーにフォルダ構造を表示し、クリックで移動できる |
| サムネイル一覧 | フォルダ内のファイルをグリッド形式でサムネイル表示する |
| ネイティブD&D | サムネイルをドラッグして他のアプリ（エクスプローラー・フォトショップ等）にドロップできる。チェック済みファイルは複数一括ドロップ対応 |
| タグ管理 | ファイルに任意のタグを付与・削除・一覧できる |
| メタデータ表示 | EXIF情報（撮影日時・カメラ情報など）やファイルサイズ等を表示する |
| 検索・フィルタリング | ファイル名・タグ・日付・形式などで絞り込む |
| スライドショー | フォルダ内の画像を自動順次再生する（間隔設定可能、デフォルト10秒）。切り替え前に次の画像の深度マップをバックグラウンド計算する |
| パララックスビューア | 🌊ボタンで起動。AIによる深度推定（depth-anything-v2-small、WebGPU優先）とWebGLで視差効果を実現 |

### 画像ビューア

| 機能 | 説明 |
|------|------|
| ズームイン・アウト | マウスホイール・キーボードで拡大縮小する |
| 回転・反転 | 時計回り・反時計回り回転、左右・上下反転 |
| フルスクリーン表示 | 画面いっぱいに画像を表示する |
| 前後ナビゲーション | 同フォルダ内の前後画像に移動する |
| フィット表示 | ウィンドウサイズに合わせて自動調整 |

### 動画プレーヤー

| 機能 | 説明 |
|------|------|
| 再生・一時停止 | スペースキーまたはボタンで切り替え |
| シーク | プログレスバーまたはキーボードで指定位置に移動 |
| 再生速度変更 | 0.25x / 0.5x / 1x / 1.5x / 2x |
| ループ再生 | 単体ループ・フォルダ内連続再生 |
| 音量調整 | スライダーまたはキーボードで調整 |
| コマ送り | 一時停止中に `,` / `.` キーで1フレームずつ移動 |
| フレームキャプチャ | `S` キーまたは 📷 ボタンで現在フレームをJPEGとして保存（動画と同フォルダに出力） |

## キーボードショートカット（重要）

マウスなしで主要操作が完結できること。最低限以下を実装する。

| キー | 動作 |
|------|------|
| `←` / `→` | 前の画像 / 次の画像 |
| `↑` / `↓` | フォルダ間移動（オプション） |
| `Space` | 再生・一時停止（動画）/ スライドショー開始・停止 |
| `F` | フルスクリーン切り替え |
| `+` / `-` | ズームイン・アウト |
| `0` | ズームリセット（フィット表示） |
| `R` | 右回転 |
| `Shift+R` | 左回転 |
| `Ctrl+F` | 検索 |
| `Esc` | フルスクリーン解除 / モーダルを閉じる |
| `Delete` | ファイルをゴミ箱へ移動（確認ダイアログあり） |

## UIデザイン方針

- **テーマ**: ダークテーマを基本とする（背景色: `#1a1a1a` 〜 `#2d2d2d` 系）
- **アクセントカラー**: 青系（`#4a9eff` 程度）を使用
- **レイアウト**:
  - 左サイドバー: フォルダツリー
  - 中央メイン: サムネイル一覧 / ビューア
  - 右パネル（任意）: メタデータ・タグ
- **フォント**: システムフォントを優先。UI文字は日本語対応必須
- **アニメーション**: 最小限。遷移は快速であること

---

## パフォーマンス要件

- **大量ファイル対応**: 1フォルダに数千枚の画像があっても快適に動作すること
- **仮想スクロール**: サムネイル一覧は仮想化（表示領域外はレンダリングしない）
- **サムネイルキャッシュ**: 生成したサムネイルをディスクキャッシュして再起動後も再利用する
- **遅延読み込み**: 画像のフルサイズデータはビューア表示時にのみ読み込む
- **バックグラウンド処理**: サムネイル生成はメインスレッドをブロックしないこと

## 将来対応予定の機能（現時点では実装しない）

- **AIタグ付け**: 画像内容をAIで解析し自動タグを付与する
- **フェイス認識・グルーピング**: 人物を認識し自動アルバム作成する

これらは拡張可能な設計にしておくが、今は実装しない。

## 実装済みのAI機能

| 機能 | 実装内容 |
|------|---------|
| 深度推定 | `onnx-community/depth-anything-v2-small`（@huggingface/transformers v3、WebGPU/WASMバックエンド） |
| パララックス描画 | WebGL1 + OES_texture_float（Float32深度テクスチャ）+ 5-tap minimum depthシェーダー |
| 深度キャッシュ | メモリ内Map（セッション中）。スライドショー事前計算とParallaxViewerで共有 |

## 技術スタック

**対象OS**: Windows のみ

### 採用構成: Tauri 2 + React + TypeScript

| レイヤー | 技術 |
|----------|------|
| フレームワーク | Tauri 2 |
| バックエンド | Rust |
| フロントエンド | React 18 + TypeScript |
| スタイリング | Tailwind CSS |
| 状態管理 | Zustand |
| 仮想スクロール | TanStack Virtual |
| DB（タグ・キャッシュ） | SQLite（`rusqlite`） |

### 採用理由

- Electronより軽量（メモリ使用量 30〜80MB 程度）
- RustバックエンドでサムネイルCPU処理・ファイルスキャンが高速
- フロントエンドはTypeScript/Reactで実装できる
- 将来のAI機能（フェイス認識・タグ付け）もRustクレートまたはPythonサブプロセスで拡張可能

### 主要Rustクレート

| 用途 | クレート |
|------|----------|
| 画像デコード・サムネイル生成 | `image` |
| EXIFメタデータ抽出 | `kamadak-exif` |
| SQLiteアクセス | `rusqlite` |
| ファイル監視 | `notify` |
| 非同期ランタイム | `tokio` |

## 開発規約

### コード品質
- TypeScript（またはRust）の型を厳密に使うこと
- コンポーネントは単一責任の原則に従い小さく保つこと
- 副作用は明示的に管理すること

### テスト
- テストは実際の機能を検証すること（`expect(true).toBe(true)` は禁止）
- ハードコードでテストを通す実装は禁止
- 境界値・異常系・エラーケースも必ずテストすること

### ファイル・削除操作
- ファイル削除は必ずゴミ箱経由にすること（完全削除は避ける）
- 破壊的操作の前は確認ダイアログを表示すること
